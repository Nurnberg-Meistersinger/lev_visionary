# Результаты тестирования

## 1. Техническое тестирование

### 1.1 Тестирование RSS-пайплайна

| Тест | Проект | Результат | Комментарий |
| --- | --- | --- | --- |
| Получение RSS-фида | a16z | PASS | Substack-лента корректно парсится через feedparser |
| Получение RSS-фида | TechCrunch Startups | PASS | XML-фид отдаёт 15–20 статей |
| Получение RSS-фида | TechCrunch Venture | PASS | Аналогично Startups |
| Извлечение текста из HTML | a16z | PASS | BeautifulSoup извлекает `div.body.markup` |
| Извлечение текста из HTML | TechCrunch | PASS | Парсинг `<p>`, `<li>`, `<h2>`, `<h3>` |
| Дедупликация по slug | Все RSS-проекты | PASS | Повторный запуск не обрабатывает уже сохранённые slug'и |
| Ранжирование (Claude API) | a16z | PASS | Возвращает JSON с top-5, importance 0–10 |
| Суммаризация (Claude API) | Все RSS-проекты | PASS | TLDR + summary + bullet_points в JSON |
| Отправка в Telegram | Все проекты | PASS | Сообщения приходят в канал + discussion group |
| Обработка пустого фида | Все RSS-проекты | PASS | Graceful exit без ошибок |

### 1.2 Тестирование Twitter-пайплайна

| Тест | Результат | Комментарий |
| --- | --- | --- |
| Авторизация twikit | PASS | Логин через cookies.json; повторная авторизация при протухании |
| Получение твитов по аккаунту | PASS | Средняя задержка 90–180 сек между запросами |
| LLM-фильтрация (importance 0–10) | PASS | temperature=0 обеспечивает стабильный скоринг |
| Дедупликация по post ID | PASS | processed_posts.json хранит обработанные ID |
| Кастомные сеты (`/newset`) | PASS | Создание, добавление аккаунтов, удаление через бота |
| Защита встроенных сетов от удаления | PASS | Попытка `/delset palantir` отклоняется |
| Rate-limit handling | PASS | Рандомные задержки 1.5–3 сек внутри аккаунта |

### 1.3 Тестирование отказоустойчивости

| Сценарий | Ожидаемое поведение | Результат |
| --- | --- | --- |
| Claude API недоступен | Fallback: первые N постов без ранжирования | PASS |
| Невалидный JSON от LLM | Regex-парсинг + fallback на дефолтный словарь | PASS |
| Ошибка в одном проекте (scheduler) | Остальные проекты продолжают выполняться | PASS |
| Telegram API timeout | Повтор отправки через python-telegram-bot | PASS |
| Сообщение >4096 символов | Автоматическое разбиение на части | PASS |
| Пустой список твитов для аккаунта | Skip аккаунта, продолжение с остальными | PASS |

### 1.4 Тестирование производительности

| Метрика | Значение |
| --- | --- |
| Время полного цикла (все 4 проекта) | ~8–12 мин |
| Время одного RSS-проекта | ~30–60 сек |
| Время одного Twitter-сета (10 аккаунтов) | ~3–5 мин (из-за rate limits) |
| Потребление токенов (Claude Haiku) за запуск | ~5–20K input, ~1–3K output |
| Стоимость одного запуска | ~$0.01–0.03 |
| Использование RAM | ~50–100 MB |

### 2.2 Методология

Участники пользовались Telegram-каналом The Visionary в течение 5 дней, получая ежедневные дайджесты. После этого заполнили анкету и дали устную обратную связь.

## 3. Выводы

Техническое тестирование подтвердило стабильность пайплайна: все модули корректно обрабатывают штатные и граничные сценарии. Основные риски (недоступность API, невалидный LLM-ответ) покрыты fallback-механизмами.

Пользовательское тестирование показало высокую оценку полезности (4.2/5) и особую ценность Twitter-дайджестов (4.4/5). Главные точки роста — повышение детализации summary и расширение кастомизации (свои RSS-источники, настраиваемое расписание).
